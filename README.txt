REST API для сервиса отпусков
Описание проекта
Это проект REST API для управления местами отпуска и списками желаемого для пользователей. 
API предоставляет функционал для добавления, просмотра и управления данными мест, а также 
позволяет пользователям сохранять понравившиеся места в список желаемого (wishlist). 
Проект создан на основе Laravel и использует Docker для контейнеризации.
И перед тем как вы приступите хотел бы сказать, что так как это мой первый 
проект с использованием подобной архитектуры и технологий, возникло несколько трудностей:

Ошибки в Postman:

500 Internal Server Error — возникала из-за отсутствия корректных связей в моделях. 
Для ее исправления я проверил лог-файлы Laravel и обнаружил, что в модели User 
не была определена связь с моделью Wishlist. После добавления этой связи ошибка исчезла.

404 Not Found — возникала при попытке отправить запросы по неправильным URL-адресам.
Я проверил список маршрутов с помощью команды php artisan route:list и убедился, что 
URL-адреса в запросах верны.

Окончательно не смог довести приложения до совершенства так как потратил много времени 
именно на понимание технологий.

Как запустить проект
Требования
Docker и Docker Compose должны быть установлены на вашей машине.
Postman для тестирования API-запросов.
Шаги по установке и запуску
Клонируйте репозиторий на вашу машину:

bash
Копировать код
git clone <URL_вашего_репозитория>
Перейдите в каталог проекта:

bash
Копировать код
cd rest-api-vacation-service
Поднимите контейнеры с помощью Docker Compose:

bash
Копировать код
docker-compose up -d
Запустите миграции для создания таблиц в базе данных:

bash
Копировать код
docker-compose exec app php artisan migrate
Проверьте, что приложение работает, выполнив команду:

bash
Копировать код
docker-compose exec app php artisan serve
Приложение будет доступно по адресу: http://localhost:8000.

Для тестирования используйте Postman. Примеры запросов описаны ниже.

Структура проекта
Проект реализует следующие функциональные возможности:

Управление местами отпусков:
Получение списка всех мест.
Добавление нового места с его координатами.
Список желаемого (Wishlist):
Получение списка мест в wishlist для конкретного пользователя.
Добавление места в wishlist пользователя (с ограничением не более 3 мест).
Примеры запросов API
1. Получение всех мест
Метод: GET

URL: http://localhost:8000/api/places

Пример ответа:

json

[
  {
    "id": 1,
    "name": "Beach Resort",
    "latitude": "25.276987",
    "longitude": "55.296249"
  }
]
2. Добавление нового места
Метод: POST

URL: http://localhost:8000/api/places

Тело запроса:

json

{
  "name": "Beach Resort",
  "latitude": "25.276987",
  "longitude": "55.296249"
}
Пример ответа:

json

{
  "message": "Место добавлено успешно",
  "place": {
    "id": 1,
    "name": "Beach Resort",
    "latitude": "25.276987",
    "longitude": "55.296249"
  }
}
3. Получение wishlist пользователя
Метод: GET

URL: http://localhost:8000/api/users/1/wishlist

Пример ответа:

json

[
  {
    "id": 1,
    "place_id": 1,
    "user_id": 1,
    "place": {
      "name": "Beach Resort",
      "latitude": "25.276987",
      "longitude": "55.296249"
    }
  }
]
4. Добавление места в wishlist
Метод: POST

URL: http://localhost:8000/api/users/1/wishlist

Тело запроса:

json

{
  "place_id": 1
}
Пример ответа:

json

{
  "message": "Место добавлено в список желаемого"
}
